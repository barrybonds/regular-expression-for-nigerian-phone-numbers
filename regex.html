Tested Number Formats

p = passed

  1. +234 8075720373 p
  2. +2348075720373 p
  3. 234 8075720373 p
  4. 2348075720373 p
  5. 08075720373 p
  6. 080 75720373 p

The reg ex code

[0-9]{3}\s[0-9]{8}|[0-9]{11}|[234]{3}\s[0-9]{10}|[+234]{4}\s[0-9]{10}|[+234]{4}[0-9]{10}|[234]{3}[0-9]{10}

How to use

Copy and Paste the reg ex code into the pattern attribute of your HTML5 text input tag

What's the use

Client side validation for Nigerian phone numbers therefore sever side validation becomes a fallback option

The Discovery

On going through a database containing over 20k Nigerian phone numbers, I discovered that the above "Test Number Formats" were the most common ways phone numbers were inputed. I also discovered that even after showing users an accepted input format,formating code was usually buggy and they could accept other formats as well. In many cases, users didn't know to add a "+" before starting with country code.

The Phyilosophy

Since there is really no standard way of writing Nigerian phone numbers, It would be nice if our forms accepted most of the ways Nigerians wrote their numbers especially when an app is used by only people with Nigerian numbers.
